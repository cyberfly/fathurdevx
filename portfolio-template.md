# Portfolio Page Template

This template provides a comprehensive structure for creating engaging portfolio pages that highlight problems, solutions, and real-world impact.

## Frontmatter Structure

```yaml
---
title: "[Project Name] - [Brief Descriptor]"
slug: project-slug
date: YYYY-MM-DD
description: "[One-line description highlighting the core value proposition and target use case]"
category: [App|Game|Training|Service]
heroImage: /images/project-hero.jpg
tags:
  - Tag 1
  - Tag 2
  - Tag 3
liveUrl: https://project-url.com
sourceUrl: https://github.com/username/repo (optional)
order: 1
challenge: "[2-3 sentences describing the main problem users face before your solution. Be specific about pain points.]"
solution: "[2-3 sentences explaining how your project solves the problem. Focus on key innovations and tangible outcomes.]"
---
```

## Writing Tips

### Problem Section
- Be hyper-specific about pain points
- Include actual quotes (or realistic ones)
- Quantify frustrations when possible
- Make it relatable to target users

### Solution Section
- Show the transformation clearly
- Use step-by-step workflows
- Include actual examples with real data
- Demonstrate the "before and after"
- Focus on user benefits, not just features

### User Journey
- Use code blocks for step-by-step clarity
- Include actual example inputs
- Show progressive states
- Make it scannable
- Demonstrate the complete flow

### Impact Section
- Always quantify when possible
- Show percentage improvements
- Compare before/after scenarios
- Include time savings
- Highlight error reduction

### General Guidelines
- Keep paragraphs short (2-4 sentences max)
- Use bullet points liberally
- Include specific examples throughout
- Write in second person for user benefits ("you get", "your workflow")
- Write in first person for development insights ("I tested", "I built")
- Balance technical depth with accessibility

## Example Category Variations

### For Apps
- Focus on user workflows and pain points
- Emphasize time savings and convenience
- Show mobile/cross-platform benefits

### For Games
- Highlight engagement and motivation
- Show progression systems
- Emphasize fun factor and learning

### For Training/Courses
- Focus on learning outcomes
- Include participant numbers/reach
- Show practical applications
- Highlight methodology

### For Services
- Emphasize results and ROI
- Show client success stories
- Highlight process and approach
- Include testimonials or outcomes
